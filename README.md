# baidu-qiandao

一个集成百度贴吧自动签到、12306 余票查询/辅助购票、大麦库存查询以及本地大模型对话的 Windows 桌面工具。项目同时提供命令行入口和图形界面入口，图形界面基于 CustomTkinter，支持托盘、定时任务、开机自启和本地配置持久化。

## 主要功能

### 百度贴吧签到

- 支持通过 BDUSS 登录态进行贴吧签到。
- 支持获取关注贴吧列表，并跳过当天已签到的贴吧。
- 支持逐个签到，失败时自动重试。
- 支持一键签到；若一键签到因权限或接口限制失败，可回退到逐个签到。
- 支持每日定时签到，可选择“一键签到”或“逐个签到”模式。
- 支持签到日志展示，便于查看成功、失败和异常原因。

### 12306 余票查询与辅助购票

- 支持粘贴 12306 登录 Cookie 并校验登录状态。
- 支持按出发站、到达站、日期、车次类型、席别查询余票。
- 支持轮询查询，可设置查询间隔和定时开抢时间。
- 支持加载常用乘车人，用于自动下单流程。
- 支持 Dry-run 测试模式，用于验证下单前置流程。
- 支持调用浏览器打开 12306 登录页或购票查询页，便于人工完成验证、选车次和支付。

### 大麦库存查询

- 支持粘贴大麦登录 Cookie 并校验关键 Cookie 是否存在。
- 支持通过演出 URL 或 itemId 查询演出详情和库存信息。
- 支持立即查询或定时轮询。
- 支持调用浏览器打开演出页面，命中库存后由用户手动完成下单。

### 本地大模型对话

- 支持连接本机 Ollama 服务，默认地址为 `http://localhost:11434`。
- 支持添加、保存、删除多个模型服务地址。
- 支持刷新模型列表并选择模型对话。
- 支持兼容 OpenAI 风格接口的其他模型服务。
- 支持 Markdown 表格和图片内容的基础渲染。

### 桌面体验

- 提供 Win11 风格图形界面。
- 支持浅色、深色、跟随系统主题切换。
- 支持最小化到系统托盘。
- 支持开机自启，并可选择自启后自动缩小到托盘。
- 配置文件损坏时会自动备份并重建默认配置。

## 运行方式

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动图形界面

```bash
python gui.py
```

### 启动命令行签到

```bash
python main.py
```

### 打包 Windows 程序

项目提供 `build.ps1`，可通过 PyInstaller 和 Inno Setup 打包：

```powershell
.\build.ps1
```

也可以只构建 exe：

```powershell
.\build.ps1 -SkipInstaller
```

## 配置说明

首次运行时会自动生成 `config.json`。也可以参考 `config.example.json` 手动创建。

常用配置项：

- `bduss`：百度账号登录态，用于贴吧签到。
- `sign_interval`：逐个贴吧签到时的间隔秒数。
- `timeout`：网络请求超时时间。
- `max_retries`：单个贴吧签到失败后的最大重试次数。
- `proxy`：代理地址，可留空。
- `schedule_enabled`：是否启用每日定时签到。
- `schedule_time`：定时签到时间，例如 `08:00`。
- `schedule_mode`：定时签到模式，`onekey` 表示一键签到，`normal` 表示逐个签到。
- `startup_minimize_to_tray`：开机启动后是否最小化到托盘。
- `12306_query_interval`：12306 余票轮询间隔。
- `12306_auto_order_enabled`：是否启用 12306 自动下单流程。
- `12306_auto_order_dry_run`：是否启用 Dry-run 测试模式。
- `chrome_path`：Chrome 或 Edge 浏览器可执行文件路径，自动检测失败时可手动填写。
- `ollama_endpoints`：大模型服务地址列表。
- `ollama_current_endpoint`：当前使用的大模型服务地址。

## 注意事项

1. `config.json` 中可能包含 BDUSS、12306 Cookie、大麦 Cookie 等敏感信息，请勿上传到公开仓库，也不要发送给他人。
2. BDUSS 和 Cookie 等同于账号登录凭证，泄露后可能导致账号被他人登录或操作。
3. 百度、12306、大麦等平台接口可能变化，若接口返回异常、校验失败或页面结构变更，功能可能需要更新代码后才能继续使用。
4. 12306 和大麦都有风控策略，频繁请求、频繁切换 IP、多设备同时登录或使用自动化操作，可能触发验证、限制或账号风险。
5. 12306 自动下单功能存在违反平台规则的风险，建议优先使用 Dry-run 测试模式；真实下单后仍需在规定时间内手动支付。
6. 大麦部分场景人机验证较强，本项目主要用于库存查询和打开浏览器辅助操作，不保证能自动完成下单。
7. 定时任务依赖程序持续运行；如果程序退出、电脑休眠或网络断开，定时签到/查票不会执行。
8. 使用代理时请确保代理稳定可靠，否则可能导致登录态失效、请求超时或触发平台风控。
9. 打包前请确认依赖已安装完整；如果需要生成安装包，还需要本机安装 Inno Setup。
10. 本项目仅供个人学习和自用，请勿用于黄牛、倒卖、批量注册、批量抢票或其他违规用途。
